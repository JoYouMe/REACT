1. React 사용 이유
    - 최근 몇 년간 자바스크립트에 대한 개발이 활발하게 이뤄지고 있음
    예전에는 브라우저에서 간단한 연산이나 시각적인 효과를 줄 때만 자바스크립트를 사용했지만
    현재는 화면단(React, Vue, Angular)뿐만 아니라 서버단(node)까지 확장하여 사용
    기존의 화면단은 데이터가 변할 때마다 화면단에 변화를 줬는데, React는 기존 화면단(뷰)를 날려버리고
    새로운 화면단(뷰)을 렌더링 하는 방식
    - React는 오직 뷰(화면단)만 신경쓰는 라이브러리
    - React에서 화면의 특정 부분을 정하는 선언체가 존재하는데 이것을 컴포넌트라고 함
    - 컴포넌트는 화면의 부분을 나눠서 만듦
    - 컴포넌트 사용 시 코드의 재사용률이 상승
    - React에서는 렌더링(사용자에게 화면을 표출하는 메소드)이라는 개념을 사용
    - 렌더링은 화면이 처음 표출되는 초기 렌더링과 데이터가 변경됐을 때 다시 실행되는 리렌더링으로 구분
    - 렌더링은 render()라는 메소드로 실행되게 됨

2. React 특징
    - Virtual DOM(가상 돔)
      리렌더링 시 새로운 뷰가 바로 DOM에 적용 되지 않음 
      새로 생성된 뷰는 Virtual DOM에 렌더링되고 이전에 생성된 Virtual DOM과 비교
      두개의 Virtual DOM의 변화된 부분만 실제 DOM에 적용
